Общее описание и назначение

Платформа TaskVizor предназначена для автоматизации работы и контроля “полевых сотрудников”, то есть сотрудников, которые значительную часть рабочего времени проводят вне офиса. Она включает в себя рабочее место диспетчера (оператора/руководителя) и мобильное рабочее место выездного сотрудника.

Первое из них описывается в данном руководстве.

Рабочее место диспетчера представляет собой портал управления, доступный для использования на любом компьютере с установленным браузером и доступом к интернет-соединению.

В веб-интерфейсе происходит управление тремя основными процессами:

  • Планированием маршрутов и заданий
  • Мониторингом - оперативным контролем
  • Аналитикой - контролем и анализом результатов на основе предоставляемых отчетов

Режим Планирование

В этом режиме область экрана разделена на три части:

  • Слева область точек, доступных для построения маршрутов и постановки заданий
  • В середине область с маршрутами, где их можно создавать, редактировать и наполнять заданиями
  • Справа карта, отображающая маршруты и точки посещения

Создание маршрутов и заданий

Создание маршрута

Для добавления нового маршрута нажмите на кнопку .

В появившемся окне введите необходимые значения полей.

  • “Ответственный” - сотрудник, на которого назначается выполнение маршрута.
  • “Дата начала маршрута” и “дата окончания маршрута” - интервалы выполнения маршрута.
  • “Адрес отправления” - используется, если будет происходить оптимизация порядка посещений внутри маршрута, при этом первая точка должна быть закреплена (например, офис или место жительства сотрудника).
  • “Повторяющийся маршрут” - используется для задания правил повторения маршрута.

После заполнения полей щелкните по кнопке “ОК”.

Повторяющиеся маршруты

Как видно из названия маршруты могут повторяться по определенным доступным правилам повторения. Для того чтобы создать повторяющийся маршрут дополнительно необходимо в окне создания маршрута:

  1. Установить галочку “Повторяющийся маршрут”;
  2. Выбрать правило повторения;
  3. Указать даты “Начала действия правила” и “Окончания действия правила” - временного интервала, по которому будут созданы маршруты-копии.

Для повторяющихся маршрутов доступны следующие правила:

  1. “Каждую неделю в определенные дни”;
  2. “Указанные дни месяца”.

Внимание!!! Для повторяющихся маршрутов интервал начала и конца маршрута может быть назначен в пределах одного дня, т.е. правила применяются только для однодневных маршрутов.

Для правила “Каждую неделю в определенные дни” маршрут будет повторяться каждую неделю в отмеченные дни. Например, еженедельно по Вт и Чт.

Для правила “Указанные дни месяца” интервал разбит на 4 недели, дни повторения указываются для каждой из четырех недель отдельно. Например, ежемесячно по 1 и 3 неделям в Пн, по 2 и 4 неделям в Ср.

Особенностью работы с повторяющимися маршрутами является возможность одним действием редактировать серии маршрутов.

При добавлении или удалении заданий в повторяющихся маршрутах, пользователю будет предложено применить изменения ко всем будущим маршрутам. При подтверждении набор точек обновленного маршрута заменит наборы точек всех будущих маршрутов в рамках одной серии.

Добавление посещений перетягиванием

Для добавления планируемых посещений в маршрут, можно воспользоваться двумя опциями - перетягиванием из области точек либо созданием внутри маршрута. Рассмотрим первый вариант.

Для добавления одной или нескольких точек в маршрут выделите их в находящейся слева области. При этом Вы можете пользоваться фильтрами, сужающими список точек. На рисунке показан пример применения фильтра по названию точки.

После выделения щелкните левой кнопкой мыши по строке с любой из выделенных точек и, не отпуская левую кнопку, передвиньте указатель мыши в среднюю область. Отпустите левую кнопку мыши, когда указатель будет находиться над нужным маршрутом.

При добавлении точек в маршрут по одной - предварительно их можно не выделять, а сразу начать перетягивать.

При этом если некоторые точки не привязаны к сотруднику, то система предложит привязать их на “лету”. Подробнее с эти можно ознакомиться здесь и здесь.

Добавление заданий “на лету”

Добавить точку можно и внутри маршрута. Этот способ удобен тем, что сразу же можно и назначить на точку задание, а при необходимости - добавить новую точку и даже нового клиента. Минус данного способа в том, что так можно добавлять точки только по одной.

Для добавления щёлкните по кнопке   в строке маршрута:

В появившемся окне Вы можете найти нужную точку по её названию, адресу либо клиенту.

Если нужной точки нет,  её можно добавить в базу нажатием на  справа от точки. При этом раскроется дополнительная часть окна с реквизитами точки. Подробнее о процессе создания точек здесь.

После выбора точки или добавления новой определите для посещения нужный вид и тип работ и при необходимости приоритет. Подробнее о назначении типа работ здесь.

Управление заданиями

Чтобы просмотреть маршрут детально, щелкните по нему. Раскроется список всех точек маршрута, а на карте отобразятся их координаты.

Редактирование задания. Назначение типа работ и приоритета

Для того, чтобы добавить к посещению точки шаблон задания, щелкните по кнопке “Редактировать задание”.

В появившемся окне выберите вид работ, тип работ и при необходимости приоритет и описание.

После нажатия на кнопку “ОК” тип работ сохранится и задание появится в мобильном приложении сотрудника

Автоматическое назначение типа работ для заданий

В системе присутствует возможность привязывать к точке “Тип работ” по умолчанию. Это позволяет автоматически назначать “Тип работ” при добавлении задания.

Для того чтобы настроить автоматическое назначение типа работ необходимо выполнить следующие действия:

  1. Настроить типы точки.

“Тип работ” привязывается к типу точки. По этому можно настроить существующие или добавить новые типы точек. Для этого перейдите “Справочники” -> “Атрибуты точек” -> “Типы”. Добавьте новый () или отредактируйте существующий () тип точки. Укажите один из доступных типов работ.

 

  1. Назначить точкам ”тип” с указанным “типом работ”.

Данная процедура производиться в меню создания и редактирования точек. Установите в поле “Тип” Ваш тип точки с установленным “типом работ”.

После выполнения указанных действий, при следующем добавлении точки в маршрут будет установлен тип работ по умолчанию.

Режим Мониторинг

В данном режиме осуществляется контроль за сотрудниками в реальном времени. В левой части расположен список сотрудников, в правой - карта.

На карте сотрудники будут отображаться, если с помощью галочки выделить нужные строки в списке. По щелчку на верхней галочке можно отображать и скрывать всех сотрудников. Далее в колонках списка могут быть отображены:

  • Сотрудник - фамилия и имя;
  • Адрес - ближайший адрес к координатам сотрудника на момент последнего сообщения от мобильного приложения-трекера;
  • Связь - , если последние данные приходили менее 5 минут назад;  - если более 5 минут;
  • Заряд - уровень заряда аккумулятора на момент последнего сообщения;
  • Процент - процент выполненных заданий по текущему маршруту (если на текущую дату попадает несколько маршрутов, задания по ним суммируются);
  • Чат - вызов чата для отправки сообщений данному сотруднику, окно с чатом появляется в правом нижнем углу;

По умолчанию список отсортирован по имени сотрудника, но Вы можете изменять сортировку, щелкая по заголовкам нужных столбцов. Например, так выглядит сортировка по проценту выполненных заданий.

При подведении курсора к имени сотрудника отображается всплывающая подсказка с подробной информацией, в том числе с временем последнего сообщения. Также подсказка появляется по щелчку на значке сотрудника на карте.

По щелчку на значении процентов справа от списка открывается окно заданий. В нём Вы можете увидеть все задания по текущему маршруту (или нескольким маршрутам, если они попадают на сегодняшнюю дату), а также статусы заданий. По щелчку на определенном задании карта центрируется на точке-адресе его выполнения.

Мониторинг выполнения заданий

Режим Отчеты

Построение отчетов

Для построения отчета выберите его тип, интервал дат и сотрудников (одного или нескольких), и нажмите кнопку “Выполнить”. Результаты отобразятся в правой нижней части экрана.

Треки и результаты посещений точек по GPS обновляются один раз в день (ночью) и данные становятся доступными на утро следующего дня.

Обратим внимание, что если по умолчанию результаты сгруппированы, отобразить отчет детально можно по нажатию кнопок группировки  на соответствующем уровне.

Далее рассмотрим различные типы отчетов по отдельности.

Сводный отчет

В сводном отчете отражена информация общего характера, сгруппированная при этом по маршрутам (если маршруты создаются на день, то и группировка будет по дням)

В отчёт могут входить следующие колонки:

  • “Сотрудник”. Фамилия и имя.
  • “Начало маршрута”, “Конец маршрута”. Для однодневных маршрутов это будет одна и та же дата, для многодневных - разные.
  • “Начало дня”. Время посещения (входа) первой точки.
  • “Конец дня”. Время выхода из последней точки.
  • “Точек в маршруте”. Общее количество точек в маршруте.
  • “По маршруту”. Количество точек маршрута, посещение которых было зафиксировано по GPS.
  • “Не по маршруту”. Количество точек, посещение которых было зафиксировано по GPS, но точки не были в маршруте. Речь идет только о точках, привязанных к данному сотруднику.

  • “Рабочее время”. Время между началом и концом дня. Если маршрут многодневный, происходит суммирование по каждому дню.
  • “На маршрутных точках”. Время, проведенное на точках, входящих в маршрут.
  • “На всех точках”. Время, проведенное на всех точках, в том числе не входящих в маршрут.
  • “На передвижение”. Время, затраченное на передвижение между точками. Разница между рабочим временем и временем на точках.
  • “Процент на маршрутных”. Отношение времени на маршрутных точках к рабочему времени.
  • “Процент на всех”. Отношение времени на всех токах к рабочему времени.
  • “Пробег”. Путь, пройденный сотрудником (пешком или на автомобиле). Измеряется по показаниям GPS.

Базовый отчет

В базовом отчете отражена информация в разрезе каждой точки маршрута.

В отчет могут входить следующие колонки:

  • “Сотрудник”. Фамилия и имя сотрудника.
  • “Дата”. Дата, к которой относится фактическое посещение точки.
  • “Начало маршрута” и “Конец маршрута”. Относятся к маршруту, в котором было запланировано посещение этой точки. Для однодневных маршрутов это будет одна и та же дата, для многодневных - разные.
  • “Точка”. Название точки.
  • “Адрес”. Адрес точки.
  • “Длительность”. Время, проведенное на точке.
  • “GPS”. Статус посещения точки по GPS (посещено/не посещено).
  • “Форма”. Кнопка вызова результатов задания, прикрепленной к посещению точки. В зависимости от результата цвет кнопки имеет 2 цвета:

- результат формы присутствует.

- результат отсутствует.

  • “Геокодирована”. Статус того, проверены координаты точки или нет. Если координаты не проверены, статус посещения точки по GPS может быть определен неверно.

Подробный отчет

В подробном отчете отражена информация в разрезе каждого посещения точки маршрута. Отличается от базового тем, что в случае нескольких посещений одной и той же точки, в нём будет выведена информация о каждом посещении. Колонки этого отчёта совпадают с базовым, кроме того в нем добавлены конкретное время входа и выхода по каждой точке.

Группировка и сортировка столбцов отчета

При выполнении любого из типа отчетов результат группируется. По нажатию кнопок группировки  раскрываются детали соответствующего уровня. Для быстрого открытия деталей вплоть до нижнего уровня нужно нажать на кнопку группировки в шапке таблицы отчета.

В таблице отчета по каждому из столбцов возможна сортировка данных. Для применения сортировки по выбранному столбцу необходимо нажать на заголовок столбца:

После применения сортировки в заголовке столбца отобразится стрелка, указывающая направление сортировки и число, обозначающее приоритет сортировки. Числовое значение появляется при применении сортировки по нескольким столбцам. Наивысший приоритет имеет сортировка по столбцу со значением “1”.

Треки и корректировка точек

В строках отчётов содержится кнопка вызова трека на карте.

По её нажатию на карте будет выведен трек (линия движения) сотрудника за интервал выполнения отчета. При этом в базовом и подробном отчёте карта дополнительно центрируется на точке, которой соответствует указанная строка.

После отображения трека и точек на карте появляется возможность корректировки положения и радиуса точек. Перемещение точки производится за ее ярлычек ()

Работа с настройками и справочниками

Настройки

В настройках доступны следующие пункты:

  1. “Время”
  2. “Пользовательские”
  3. “Отчеты”

Внешний вид меню “Настройки”:

Каждый пункт настроек имеет “Статус блокировки”, “Глобальное(по умолчанию) значение” и “Пользовательское значение”.

“Глобальное значение” - значение, устанавливаемое по умолчанию для всех пользователей.

“Пользовательское значение” - значения, устанавливаемое для конкретного пользователя.

“Статус блокировки” - разрешает или запрещает пользователю изменять значение по умолчанию.

Видеть “Глобальное значение”, а также изменять статус блокировки могут пользователи с ролью администратор.

Настройки времени, языка, местности

На вкладке “Время” доступны следующие изменение настроек:

  • “Установка часового пояса”
  • “Язык приложения”
Настройки планирования

На вкладке “Планирование” доступны следующие изменение настроек:

  • “Отображать только свободные точки”

При установке данного флага точка может быть добавлена в маршрут только единожды. После этого она будет недоступна в области “Точки”.

  • “Маршрут готов к синхронизации”

При установке данного флага, после создания маршрута, по умолчанию, все маршруты доступны для синхронизации. В этом случае после добавления к заданию в маршруте “Типа работ” - при первой возможности задание будет синхронизировано с мобильным устройством.

В системе значки отображаются следующим образом:

- Синхронизация маршрута включена;

- Синхронизация маршрута отключена.

  • “Отображать значок отправки на email”

При отображении данного значка появляется возможность отправки маршрута на электронную почту

В системе значок выглядит следующим образом -

  • “Оптимизация маршрутов”

При добавлении данного значка появляется доступным меню оптимизации маршрутов.

В системе значок выглядит следующим образом -

Меню оптимизации маршрутов выглядит следующим образом:

  • “Отображать статус повторяющегося маршрута”

В случае отображения статуса внешний вид будет следующий:

- Маршрут повторяющийся

- Маршрут не повторяющийся

Настройки отчетов

На вкладке “Отчеты” доступны следующие изменение настроек:

  • “Длительность рабочего дня (ч)”

Указывается длительность рабочего дня. Значение используется  для расчета параметров “% на маршрутных” и “% на всех” в сводном отчете.

  • “Максимальный разрыв данных GPS (с)”

Указывается время, в течение которого выход gps-координаты за пределы точки не будет разделять посещение, при условии последующего возврата в область точки. В противном случае, при превышении указанного времени будет завершено одно посещение и, при следующем попадании gps-координаты в область точки, будет начато фиксирование нового посещения. Данная настройка используется для фильтрации возможных выбросов и погрешностей gps сигнала.

  • “Минимальная длительность посещения (с)”

Справочник “Точки”

Справочник точек выглядит аналогично другим справочникам, однако имеются некоторые особенности:

  • Справа от списка элементов находится карта
  • При добавлении или редактировании точки работа ведётся с более детализированным видом, в том числе взаимодействующим с картой

Расположен данный справочник также отдельно от остальных, чтобы перейти к нему, щелкните во всплывающем меню ниже справочников.

При добавлении или редактировании точки появляется следующий вид.

  1. “Название точки”, обязательное поле.
  2. “Клиент”. Если клиенты не ведутся, следует оставить значение по умолчанию (неизвестно).
  3. “Кнопка геокодирования адреса”. Сам адрес вводится в п.5 По нажатию на кнопку на  производится поиск адреса с помощью поисковой системы, которая выбирается в п.4, а найденные координаты отображаются на карте.
  4. Выбор “поисковой системы”.
  5. Текст адреса для поиска.
  6. Поле, в котором отображается адрес, определенный поисковой системой после нажатия правой кнопки мыши на любой области карты.
  7. “Нормализованный адрес” (с разбиением город-улица-дом). Отображается, если точка создавалась не вручную, а путем автоматизированного переноса данных из учётной системы.
  8. “Здание”. Номер дома вместе с корпусом или другими атрибутами.
  9. “Описание к адресу”. Любая текстовая информация, помогающая определить местоположение адреса.
  10. “Подтвердить”. Статус того, проверены координаты точки или нет.
  11. “Радиус точки”. Расстояние от координат точки, внутри которого будет засчитано посещение.
  12. “Широта”.
  13. “Долгота”.
  14. “Тип”. Значение из справочника. По умолчанию неизвестно.
  15. “Статус”. Значение из справочника. По умолчанию неизвестно.
  16. “Регион”. Значение из справочника. По умолчанию неизвестно.
  17. “Описание точки”. Информация в данном поле доступна в всплывающей подсказке на вкладке “Планирование” при наведении на точку, а также может быть отображена на мобильном устройстве при выполнении задания.
  18. “Привязать точку к сотруднику”. Область привязки сотрудников к точке. Можно выбрать одного сотрудника, нескольких или всех. По умолчанию точка также привязывается к сотруднику, который ее создает.
  19. “Стартовая точка”. Настройка используется в оптимизации маршрута. В оптимизированных маршрутах точки отмеченные как стартовые будут помещаться на первое место.
  20. “Контакты”. В данном месте можно добавлять, редактировать и удалять контакты относящиеся к точке.
  21. Кнопки навигации по списку и сохранения.

“Отмена” - отменить изменения и вернуться к списку точек.

“Сохранить” - сохранить изменения и вернуться к списку точек.

“Сохранить | Далее” - сохранить изменения и перейти к следующей точке в списке.

“Далее” - перейти к следующей точке в списке без сохранения.

Для редактирования координат на карте следует щелкать на том месте карты, куда необходимо выставить координаты точки.

Справочник “Сотрудники”

В справочнике по сотрудникам присутствует информация по самим “Сотрудникам”, а также по дополнительным атрибутам, таким как “Статусы” и “Должности”

На вкладке “Сотрудники” можно просматривать список всех сотрудников, добавлять новых, редактировать и удалять существующих сотрудников, а также привязывать и отвязывать точки.

Вкладка “Сотрудники” состоит из:

  1. Блока фильтров (1). Блок фильтров позволяет быстро ориентироваться в списке сотрудников. Например поиск конкретного сотрудника по полю “Имя” или “Фамилия”, или выделение группы сотрудников, объединенной общим параметром, например по полю  “Руководитель”.
  2. Блок со списком сотрудников (2).

При добавлении сотрудника появляется следующий вид.

  1. “Фамилия” Обязательное поле;
  2. “Имя”. Обязательное поле;
  3. “Отчество”;
  4. “Должность”. По умолчанию “Неизвестно”, добавляется на вкладке “Должности”;
  5. “Руководитель”. По умолчанию не выбрано ничего. В качестве руководителя можно выбрать любого из существующих сотрудников; После назначения руководителя данный сотрудник становиться доступен в учетной записи руководителя (Руководитель для подчиненного сотрудника в рамках прав доступа может создавать/просматривать/редактировать/удалять маршруты, назначать/снимать точки и т.д.);
  6. “Дата рождения”;
  7. “Система gps-мониторинга”
  8. “Внешний код”, произвольное значение для идентификации пользователя.
  9. “Код пользователя”, предназначено для связки сотрудника с пользователем системы.

Привязка точек к сотруднику

В отчетах анализ посещений точек (по gps) производить только по точкам, которые привязаны к сотруднику. Поэтому прежде чем выполнять отчеты необходимо проверить привязаны ли к сотруднику точки или нет.

Открытие окна для привязки точек производится нажатием соответствующей точки “Привязка точек” 

Окно привязки точек выглядит следующим образом:

Левой часть -  список точек доступных для добавления, справа - список точек уже добавленных к сотруднику. Сверху обоих списков присутствует блоки фильтров, которые позволяют упростить поиск требуемой точки или группы точек.

Процесс привязки (отвязки) точек включает в себя следующие шаги:

  1. Из списка доступных (уже добавленных) точек отметить необходимые. Для быстрого поиска требуемой точки или набора точек можно воспользоваться фильтрацией.
  2. Нажать кнопку “Назначить” (“Снять”). Отмеченные точки переместятся в список доступных (недоступных) к сотруднику.

Сохранения состояния происходит каждый раз после нажатия кнопок “Назначить” или “Снять”

Привязка точек к сотруднику “на лету”

Привязки точек “на лету” происходит при добавлении точек в маршрут. То есть если при добавлении точки в маршрут она не привязана, то появится всплывающее окно с предложением привязать точку.

Внимание!!! Если отказаться от привязки, то точка не будет добавлена в маршрут. Также если при добавлении группы точек в маршрут и часть из них уже привязана, но отказаться от привязки, то привязанные точки также не будут добавлены в маршрут.

Шаблоны заданий

Шаблоны заданий назначаются на точку в маршруте и в совокупности образуют "Задачу для выполнения”. В шаблоне заданий может быть организована последовательность действий, перечень работ для выполнения, сбор необходимой информации или любой другой произвольный объем запланированных действий.

Для того, что начать работу с “Шаблонами заданий” откройте “Справочники” -> “Шаблоны заданий”.

На вкладке шаблоны заданий доступны следующие элементы:

  1. “Формы”;
  2. “Приоритеты”;
  3. “Виды”.
  4. “Причины отмены”

В окне соответствующего элемента доступны действия по созданию, редактированию и удалению.

Формы

В окне “Формы” доступно создание, редактирование и удаление форм.

Окно условно  можно разделить на три части:

  1. Центральная часть, предназначенная для создания, редактирования и удаления форм заданий;
  2. Левая часть - полный список элементов из которых формируется форма;
  3. Правая часть - отображает свойства отмеченного элемента формы.
